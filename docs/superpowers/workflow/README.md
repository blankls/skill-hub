## 工作流概览

```
需求分析 → 计划制定 → 开发执行 → 全面测试 → 收尾提交
   ↓          ↓          ↓          ↓          ↓
  🎯        📝        🛠️        🧪       📦
  明确目标   详细计划   逐步开发   验证质量   整理发布
```

## 阶段详解

### 1️⃣ 第一阶段：需求分析与设计

#### 1.1 需求收集与澄清
- **任务清单法**：列出所有需求点
- **用户问题**：明确目标，比如我们一开始的「页面风格优化」
- **功能确认**：逐个确认，避免歧义
- **优先级排序**：哪些是核心，哪些是次要

#### 1.2 设计文档
- 使用 `canvas-design` 或 `frontend-design` 技能
- 确认视觉风格、配色方案
- 确定交互模式和用户体验

---

### 2️⃣ 第二阶段：制定开发计划

#### 2.1 创建书面计划文档
- **必用技能**：`writing-plans`
- 文档包含：
  - ✅ 项目概览与目标
  - 📂 文件变更清单
  - 📝 分步任务
  - ⏱️ 估计时间

#### 2.2 任务分解
- 大任务拆成小任务
- 每个任务明确目标和验收标准
- 例如 Skill Hub 项目的：
  1. 全局颜色和字体系统
  2. Hero Section 重写
  3. 全局字体统一
  4. 主题切换修复
  5. 最终测试

---

### 3️⃣ 第三阶段：开发执行

#### 3.1 使用子代理驱动开发
- **必用技能**：`subagent-driven-development`
- 每个子任务独立完成
- 原子化提交，每个修改单独提交
- Git 工作区保持干净

#### 3.2 开发最佳实践
1. **修改前先读取**：确保知道当前代码的状态
2. **最小修改原则**：只改必要的
3. **立即测试**：修改后立即在浏览器查看效果
4. **热更新验证**：确保 HMR 正常工作

---

### 4️⃣ 第四阶段：全面测试

#### 4.1 测试阶段
- **必用技能**：`qa` 和 `webapp-testing`
- 测试清单：
  - ✅ 功能测试（所有按钮、表单、导航）
  - ✅ 视觉测试（样式、动画）
  - ✅ 响应式测试（不同屏幕尺寸）
  - ✅ 控制台检查（零错误）

---

### 5️⃣ 第五阶段：代码整理与提交

#### 5.1 Git 分支管理
- 保持 master 分支稳定
- feature 分支使用完后清理
- Git 工作区（worktree）管理
- 及时删除不用的分支和工作区

#### 5.2 提交规范
- **原子化提交**：一个提交一个功能
- **清晰的 commit message**：
  ```
  type(scope): description
  ```
  - **type**: feat, fix, style, perf, refactor
  - **scope**: 组件名或模块名

---

### 6️⃣ 第六阶段：收尾与发布

#### 6.1 最终状态检查
```bash
git status          # 检查工作区
git branch -a       # 检查分支
git log --oneline   # 检查提交历史
```

#### 6.2 项目文档更新
- README 文档
- 设计规范文档
- 开发日志
