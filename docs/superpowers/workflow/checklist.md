# 📋 前端开发工作流检查清单

这套完整的检查清单将帮你确保每个阶段都做到位！

---

## 1. 需求分析阶段 [ ]

- [ ] 收集所有需求点
- [ ] 明确用户目标和期望
- [ ] 按优先级排序需求
- [ ] 与用户确认理解无误

---

## 2. 计划制定阶段 [ ]

- [ ] 调用 `writing-plans` 技能创建书面计划
- [ ] 列出所有需要修改的文件
- [ ] 分解任务到可执行的小模块
- [ ] 为每个任务定义验收标准
- [ ] 估计每个任务的大致时间

---

## 3. 开发执行阶段 [ ]

- [ ] 调用 `subagent-driven-development` 技能
- [ ] **每次修改前** 先读取相关文件
- [ ] 只做**最小必要**的修改
- [ ] 修改后立即在浏览器中测试
- [ ] 每个独立修改对应一个 Git 提交
- [ ] 使用规范的 commit message：`type(scope): description`
- [ ] 保持 Git 工作区干净，不要堆积太多修改

---

## 4. 全面测试阶段 [ ]

- [ ] 调用 `qa` 技能进行全面质量检查
- [ ] 使用 `webapp-testing` 技能在浏览器中交互测试
- [ ] 逐个测试所有功能按钮
- [ ] 测试所有表单和导航
- [ ] 检查不同屏幕尺寸的响应式效果
- [ ] 查看浏览器控制台，确保**零错误**
- [ ] 如有需要，调用 `brainstorming` 优化设计

---

## 5. 收尾提交阶段 [ ]

- [ ] 检查 Git 状态：`git status`
- [ ] 检查提交历史：`git log --oneline`
- [ ] 查看所有分支：`git branch -a`
- [ ] 清理不用的 feature 分支和 Git worktree
- [ ] 调用 `finishing-a-development-branch` 技能收尾
- [ ] 检查工作区是否完全干净

---

## 6. 后续优化（可选）[ ]

- [ ] 收集用户反馈
- [ ] 迭代改进产品
- [ ] 优化性能
- [ ] 添加新功能

---

## 🎯 快速检查（每次提交前）

- [ ] 代码是否可读
- [ ] 是否遵循项目规范
- [ ] 有没有引入新的 console error
- [ ] Git diff 是否只包含必要修改
- [ ] Commit message 是否清晰
