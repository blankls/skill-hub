# 📖 前端工作流简单示例

这个示例展示了一个简单的样式优化项目。

---

## 场景

用户说：「现在的页面太难看，帮我优化一下风格。」

---

## 应用工作流

### 阶段 1：需求分析
```
Use Skill: brainstorming
```
- 目标：页面风格现代化
- 确认：保持功能，只优化视觉
- 优先级：高

### 阶段 2：制定计划
```
Use Skill: writing-plans
```
- 计划文档：列出所有修改
- 任务分解：全局颜色、字体、组件样式

### 阶段 3：开发执行
```
Use Skill: subagent-driven-development
```
- 逐个任务执行
- 每个修改单独提交

### 阶段 4：全面测试
```
Use Skill: qa
Use Skill: webapp-testing
```
- 检查所有功能
- 确认零错误

### 阶段 5：收尾提交
```
Use Skill: finishing-a-development-branch
```
- 清理分支
- 最终确认

---

## 完成！

就这么简单，任何前端项目都可以套用这个流程！🚀
