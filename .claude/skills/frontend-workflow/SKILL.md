# 🚀 前端开发完整工作流

一套完整的前端开发工作流程，基于 Skill Hub 项目实践！

## 功能

这套工作流会帮助你完成从需求到发布的完整前端开发过程：

- 📋 需求分析与澄清
- 📝 书面计划制定
- 🛠️ 子代理驱动的逐步开发
- 🧪 全面的质量测试
- 📦 代码整理与提交收尾

## 什么时候使用

当你需要：
- 从零开发一个新的前端项目
- 对现有项目进行重构或优化
- 确保开发过程规范且完整
- 使用 AI 工具提高开发效率

## 使用方法

1. **开始新项目或功能开发**
2. **按顺序使用工作流阶段**
3. **每个阶段调用对应的技能**

## 工作流阶段

### 1. 需求分析 (Brainstorming)
```
Use Skill: brainstorming
```
- 收集和澄清需求
- 确定优先级
- 明确用户期望

### 2. 制定计划 (Writing Plans)
```
Use Skill: writing-plans
```
- 创建详细的书面计划
- 分解任务和文件清单
- 定义验收标准

### 3. 开发执行 (Subagent Driven Development)
```
Use Skill: subagent-driven-development
```
- 原子化任务执行
- 每个修改单独提交
- 保持 Git 工作区干净

### 4. 全面测试 (QA)
```
Use Skill: qa
Use Skill: webapp-testing
```
- 功能测试
- 视觉和响应式测试
- 零错误验证

### 5. 收尾提交 (Finishing Branch)
```
Use Skill: finishing-a-development-branch
```
- Git 状态检查
- 分支清理
- 最终确认

## 快速检查清单

- [ ] 每个阶段都有测试和验证
- [ ] Git 提交是原子化的
- [ ] 每次修改后立即测试
- [ ] 控制台始终零错误
- [ ] 工作区保持干净
