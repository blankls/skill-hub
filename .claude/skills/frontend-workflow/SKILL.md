# 🚀 前端开发完整工作流

一套完整的前端开发工作流程，基于 Skill Hub 项目实践！适用于各种项目场景！

## 功能

这套工作流会帮助你完成从需求到发布的完整前端开发过程：

- 📋 需求分析与澄清
- 🎨 设计规划与原型
- 📝 书面计划制定
- 🛠️ 子代理驱动的逐步开发
- 🧪 全面的质量测试
- 📦 代码整理与提交收尾
- 🔄 持续迭代与优化

## 什么时候使用

当你需要：
- 🆕 **从零开发一个新的前端项目**
- 🔧 **对现有项目进行重构或优化**
- ✨ **添加新功能到现有项目**
- 🎨 **优化页面样式和用户体验**
- 🐛 **修复现有项目的 bug**
- 📱 **响应式布局或移动端适配**
- 📚 **完善项目文档和注释**
- 📊 **数据分析或图表可视化**
- 🧪 **优化性能或代码质量**
- 🚀 **部署准备或发布流程**

## 使用方法

1. **根据你的具体需求，选择对应的工作流阶段**
2. **每个阶段调用对应的技能**
3. **保持原子化修改，逐步完成**

## 工作流阶段

---

### 阶段 1：需求分析与设计 (Brainstorming & Design)

#### 1.1 需求理解
```
Use Skill: brainstorming
```
- 收集和澄清需求
- 确定优先级
- 明确用户期望

#### 1.2 设计规划
```
Use Skill: canvas-design
Use Skill: frontend-design
```
- 确定视觉风格
- 规划组件布局
- 设计交互效果

---

### 阶段 2：制定开发计划 (Writing Plans)

```
Use Skill: writing-plans
```
- 创建详细的书面计划
- 分解任务和文件清单
- 定义验收标准
- 预估开发时间

---

### 阶段 3：开发执行 (Subagent Driven Development)

```
Use Skill: subagent-driven-development
```
- 原子化任务执行
- 每个修改单独提交
- 保持 Git 工作区干净

#### 常用技能调用示例：
- 样式优化：`Use Skill: frontend-design`
- 图表展示：`Use Skill: chart-visualization`
- 数据分析：`Use Skill: data-analysis`
- 前端美化：`Use Skill: ui-ux-pro-max`
- 组件创建：`Use Skill: skill-creator`
- 浏览器操作：`Use Skill: agent-browser`

---

### 阶段 4：全面测试 (QA)

#### 4.1 功能测试
```
Use Skill: qa
```
- 功能完整性检查
- 代码质量审查
- 错误边界测试

#### 4.2 界面测试
```
Use Skill: webapp-testing
```
- 视觉和响应式测试
- 浏览器交互测试
- 零错误验证

---

### 阶段 5：收尾提交 (Finishing Branch)

```
Use Skill: finishing-a-development-branch
```
- Git 状态检查
- 分支清理
- 最终确认
- 部署准备

---

## 适用场景示例

### 场景 1：新项目从零开发
使用所有 5 个阶段，完整完成项目

### 场景 2：现有项目优化样式
使用阶段 1（需求）→ 阶段 2（计划）→ 阶段 3（开发）→ 阶段 4（测试）→ 阶段 5（收尾）

### 场景 3：修复单个 bug
简化流程：阶段 1（需求理解）→ 阶段 3（快速修复）→ 阶段 4（验证测试）→ 阶段 5（提交）

### 场景 4：添加新功能
完整流程：阶段 1（需求）→ 阶段 2（计划）→ 阶段 3（开发）→ 阶段 4（测试）→ 阶段 5（收尾）
